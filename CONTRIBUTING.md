# Contributing Guide

Awesome Developer Community in Korea에 기여해 주셔서 감사합니다! 🎉

모든 형태의 기여를 환영합니다. 새로운 커뮤니티 추가, 깨진 링크 수정, 설명 개선 등 어떤 것이든 좋습니다.

---

## 기여 방법 (How to Contribute)

### 새 커뮤니티 추가 (Add New Community)

1. 이 저장소를 **Fork** 해주세요.
2. 새 브랜치를 만들어 주세요.
   ```bash
   git checkout -b add/community-name
   ```
3. 아래 [체크리스트](#체크리스트-checklist)를 확인한 후 `README.md`를 수정해 주세요.
4. 변경사항을 커밋해 주세요.
   ```bash
   git commit -m "Add: [커뮤니티 이름] 추가"
   ```
5. **Pull Request**를 보내주세요.

### 깨진 링크 수정 (Fix Broken Link)

깨진 링크를 발견하셨다면 [Issue](https://github.com/JihunDev/awesome-developer-community-in-korea/issues/new/choose)를 열거나 직접 수정하여 Pull Request를 보내주세요.

---

## 체크리스트 (Checklist)

Pull Request를 보내기 전에 아래 항목을 모두 확인해 주세요.

- [ ] 링크가 유효하고 접근 가능한지 확인했나요?
- [ ] 적합한 카테고리에 추가했나요?
- [ ] 카테고리 내에서 **알파벳 / 가나다 순**으로 정렬했나요?
- [ ] 아래 **형식**을 따랐나요?
- [ ] 설명이 명확하고 간결한가요? (마침표 없이)
- [ ] 동일한 커뮤니티가 이미 등록되어 있지 않은지 확인했나요?

---

## 형식 (Format)

```markdown
- [커뮤니티 이름](URL) - 설명
```

### 예시 (Examples)

```markdown
- [OKKY](https://okky.kr/) - 국내 최대 Java / 개발 종합 커뮤니티
- [TensorFlow Korea](https://www.facebook.com/groups/TensorFlowKR/) - TensorFlow 한국 사용자 그룹
- [Developers Korea](https://discord.gg/kTm7xAfaNv) - 한국 개발자 종합 Discord 서버
```

### 규칙 (Rules)

| 항목 | 규칙 |
|---|---|
| **이름** | 커뮤니티의 공식 이름을 사용해 주세요 |
| **URL** | HTTPS를 지원하는 경우 HTTPS URL을 사용해 주세요 |
| **설명** | 어떤 기술 스택이나 활동을 다루는 커뮤니티인지 간결하게 작성해 주세요 |
| **마침표** | 설명 끝에 마침표(`.`)를 붙이지 않습니다 |
| **정렬** | 영문명은 알파벳 순, 한글명은 가나다 순 |

## 커뮤니티 등록 기준 (Criteria)

다음 기준을 만족하는 커뮤니티를 등록해 주세요.

- ✅ **한국 개발자**를 주요 대상으로 하는 커뮤니티
- ✅ **활성화된 커뮤니티** (최근 활동이 있는 경우)
- ✅ **건전하고 포용적인** 커뮤니티
- ✅ 기술 / 개발과 직접 관련된 커뮤니티
